# IT-SERVICE-DESK-PROJECT1
IT SUPPORT
Project Description
Managing an IT Service Desk involves more than just closing tickets; it’s about understanding patterns and optimizing response times. This project bridges the gap between raw ticketing data and strategic decision-making.

By leveraging Python for automated data engineering and Power BI for visual storytelling, this project transforms messy service logs into a streamlined performance management tool. It identifies where help desk resources are being stretched thin and highlights opportunities for automation or better staff training.

🔍 The Problem
Service desks often struggle with "data overload." With thousands of tickets coming in across different channels (email, phone, portal), it is difficult to manually track:

Which agents are overloaded.

Which departments are experiencing the most technical failures.

Whether the team is actually meeting its SLA (Service Level Agreement) targets.

💡 The Solution
This project implements a two-stage pipeline:

Data Transformation (Python): Uses Pandas to automate the "grunt work"—handling null values, calculating "Time to Resolve" in business hours, and normalizing ticket categories.

Visual Analytics (Power BI): A dynamic dashboard that allows managers to drill down from high-level yearly trends to individual ticket details with a single click.

🎯 Objective
Reduce Resolution Time: By identifying common bottlenecks in the workflow.

Resource Optimization: Helping managers allocate the right number of agents during peak ticket hours.

Proactive Maintenance: Spotting recurring hardware or software issues before they become company-wide outages.
